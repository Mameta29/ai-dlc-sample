# unit-ai ビジネスルール定義

| ID | ルール名 |
|---|---|
| BR-AI-001 | キャラクター口調一貫性 |
| BR-AI-002 | 言い訳パターン重複回避 |
| BR-AI-003 | 煽りレベル制限 |
| BR-AI-004 | ストリーミング応答タイムアウト |
| BR-AI-005 | 対話データ記録必須 |
| BR-AI-006 | プロンプトインジェクション防止 |

## BR-AI-001: キャラクター口調一貫性
- 各キャラクターのsystem promptでトーンを固定
- キャラクター切替時に即座にsystem prompt変更

## BR-AI-002: 言い訳パターン重複回避
- 直近5回の同一カテゴリ言い訳と異なる表現を生成
- previousVariationsをプロンプトに含める

## BR-AI-003: 煽りレベル制限
- 最大3段階まで（不快にならないバランス）
- ユーザーがネガティブな反応を示した場合はレベルを下げる

## BR-AI-004: ストリーミング応答タイムアウト
- 30秒でタイムアウト、エラーメッセージ表示

## BR-AI-005: 対話データ記録必須
- 全メッセージをDBに保存（unit-analyticsのデータソース）

## BR-AI-006: プロンプトインジェクション防止
- ユーザー入力はuser roleとして送信（system promptに混入させない）

---

## テスト可能なプロパティ（PBT-01準拠）

| プロパティ | カテゴリ | 説明 |
|---|---|---|
| キャラクタータイプの不変条件 | Invariant | 有効なCharacterTypeのみConversationに設定可能 |
| 言い訳カテゴリの網羅性 | Invariant | 生成される言い訳は常に有効なExcuseCategoryに属する |
| メッセージ順序の不変条件 | Invariant | 同一対話内のメッセージはcreated_atで単調増加 |

### PBT対象外
- AI生成テキスト: 非決定的出力のためPBT不適（N/A）
