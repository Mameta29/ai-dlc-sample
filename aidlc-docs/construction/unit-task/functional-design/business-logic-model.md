# unit-task ビジネスロジックモデル

## ビジネスプロセス一覧

| # | プロセス | トリガー | 関連ストーリー |
|---|---|---|---|
| 1 | タスク登録 | ユーザーがタスクを入力 | US-005, US-006 |
| 2 | タスク一覧取得 | ダッシュボード表示 | US-007 |
| 3 | タスク編集 | ユーザーがタスクを修正 | US-008 |
| 4 | タスク削除 | ユーザーがタスクを削除 | US-009 |
| 5 | タスク完了報告 | ユーザーが「完了」を押す | US-010 |
| 6 | 期限切れ検出 | 締切超過（定期チェック） | US-007 |
| 7 | 先延ばし経過率算出 | リアルタイム表示 | US-007 |

---

## BP-1: タスク登録

```
1. ユーザーがタスクタイトルを入力
2. バリデーション
   - タイトル: 1〜200文字、空白のみ不可
   - 締切: 未来日時のみ（設定する場合）
3. Taskレコード作成
   - status: PROCRASTINATING
   - started_at: 現在時刻
   - deadline: ユーザー指定 or null
4. 登録成功フィードバック表示
5. AI対話開始トリガー（unit-aiへ通知）
   - 締切ありの場合: タスク定量化質問を開始
   - 締切なしの場合: 簡易対話のみ
```

---

## BP-2: タスク一覧取得

```
1. ユーザーIDでタスクをフィルタリング
2. ステータスフィルター適用（先延ばし中/完了/期限切れ/全て）
3. ソート適用
   - デフォルト: 作成日時 降順（新しい順）
   - 切替可能: 締切日時 昇順、スコア 降順
4. 各タスクに以下を付与して返却:
   - 現在の先延ばし経過率（%）
   - 残り時間（締切ありの場合）
   - 現在の先延ばしスコア（unit-scoringから取得）
```

---

## BP-3: タスク編集

```
1. タスクID + ユーザーIDで対象タスクを取得
2. 所有権確認（自分のタスクのみ編集可）
3. バリデーション
   - タイトル: 1〜200文字
   - 締切: 未来日時のみ（変更する場合）
4. 先延ばし中のタスクの締切変更時:
   - 確認ダイアログ表示（クライアント側）
   - 先延ばし時間を新しい締切で再計算
5. 変更履歴をTaskChangeLogに記録
   - field_name: "title" or "deadline"
   - old_value / new_value を記録
6. Taskレコード更新
```

---

## BP-4: タスク削除

```
1. タスクID + ユーザーIDで対象タスクを取得
2. 所有権確認
3. 確認ダイアログ表示（クライアント側）
4. 論理削除ではなく物理削除
   - スコア確定済み（COMPLETED）の場合: 
     ランキングスコアは保持（unit-scoringのスコアは削除しない）
   - PROCRASTINATING の場合: タスクとスコア計算を削除
5. 関連するTaskChangeLogも削除（CASCADE）
```

---

## BP-5: タスク完了報告

```
1. タスクID + ユーザーIDで対象タスクを取得
2. 所有権確認
3. ステータス確認
   - PROCRASTINATING のみ完了可能
   - COMPLETED / EXPIRED は完了不可
4. Task更新
   - status: COMPLETED
   - completed_at: 現在時刻
5. unit-scoring にスコア確定をトリガー
   - finalizeScore(taskId) を呼び出し
6. 完了アニメーション/エフェクト表示（クライアント側）
7. スコアとランキングインパクトを表示
```

---

## BP-6: 期限切れ検出

```
1. 定期チェック（2つの方式を併用）
   a. クライアントサイド: タスク一覧表示時にリアルタイムチェック
   b. サーバーサイド: Cron（1時間ごと）でバッチ更新
2. deadline < 現在時刻 かつ status = PROCRASTINATING のタスクを検索
3. 対象タスクを更新:
   - status: EXPIRED
   - expired_at: deadline（締切時刻を記録）
4. 先延ばしスコアを最大値で確定
   - unit-scoring の finalizeScore を呼び出し
   - 先延ばし時間 = 100%（全期間先延ばし）
```

---

## BP-7: 先延ばし経過率算出

```
計算式:
  elapsed_percentage = (現在時刻 - started_at) / (deadline - started_at) × 100

ルール:
  - deadline が null の場合: 0% 固定（ランキング対象外）
  - elapsed_percentage > 100%: 100% にキャップ
  - status が COMPLETED の場合: completed_at 時点の値を固定表示
  - status が EXPIRED の場合: 100% 固定

クライアント側でリアルタイム計算:
  - setInterval で1分ごとに再計算
  - 画面表示: プログレスバー + パーセンテージ
```
