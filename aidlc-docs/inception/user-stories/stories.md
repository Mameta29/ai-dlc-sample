# SABOROU ユーザーストーリー

## ストーリー構成
- **分解アプローチ**: ユーザージャーニーベース
- **粒度**: 小さめ（実装タスクに近い）
- **形式**: As a [ペルソナ], I want to [行動], so that [価値]

---

# Epic 1: オンボーディング & 認証

## US-001: Googleアカウントでサインアップ
**As a** 新規ユーザー, **I want to** Googleアカウントでワンクリックサインアップしたい, **so that** 面倒な入力なしにすぐ使い始められる

**Acceptance Criteria:**
- [ ] Googleログインボタンが表示される
- [ ] OAuth認証フローが完了するとダッシュボードに遷移する
- [ ] 初回ログイン時にユーザーレコードが自動作成される
- [ ] メールアドレスとプロフィール画像がGoogleから取得される

---

## US-002: ニックネーム設定
**As a** 新規ユーザー, **I want to** ランキングに表示されるニックネームを設定したい, **so that** 本名を出さずにサービスを楽しめる

**Acceptance Criteria:**
- [ ] 初回ログイン後にニックネーム入力画面が表示される
- [ ] 2〜20文字で設定可能
- [ ] 重複チェックが行われる
- [ ] 後からプロフィールで変更可能

---

## US-003: AIキャラクター選択
**As a** 新規ユーザー, **I want to** 最初に話しかけてくるAIキャラクターを選びたい, **so that** 自分に合ったトーンで対話を始められる

**Acceptance Criteria:**
- [ ] 3種類のキャラクター（サボロー、ナマケモノ先輩、サボリスト）が紹介される
- [ ] 各キャラのサンプル発言が表示される
- [ ] 1つ選択してスタート
- [ ] 後から設定で変更可能

---

## US-004: オンボーディングチュートリアル
**As a** 新規ユーザー, **I want to** サービスの基本的な使い方を理解したい, **so that** 何をすれば得点が入るのかわかる

**Acceptance Criteria:**
- [ ] 3〜5ステップの簡易チュートリアル表示
- [ ] タスク入力→先延ばし→スコア確定の基本フローを説明
- [ ] スキップ可能
- [ ] 完了後にダッシュボードへ遷移

---

# Epic 2: タスク登録 & 管理

## US-005: テキストでタスク入力
**As a** タクヤ, **I want to** やるべきタスクをテキスト入力したい, **so that** 先延ばしの対象を登録できる

**Acceptance Criteria:**
- [ ] タスクタイトル入力フィールドが表示される
- [ ] 入力後にEnterまたはボタンで登録
- [ ] 空文字では登録できない
- [ ] 登録成功のフィードバックが表示される

---

## US-006: 締切日時の設定
**As a** タクヤ, **I want to** タスクの締切日時を設定したい, **so that** 先延ばし時間の計算基準ができる

**Acceptance Criteria:**
- [ ] 日付ピッカーで締切日を選択可能
- [ ] 時刻も設定可能（デフォルト: 23:59）
- [ ] 過去の日時は設定不可
- [ ] 締切なしも選択可能（ランキング対象外になる旨を表示）

---

## US-007: タスク一覧表示
**As a** タクヤ, **I want to** 登録済みタスクの一覧を確認したい, **so that** 何を先延ばしているか把握できる

**Acceptance Criteria:**
- [ ] 先延ばし中/完了/期限切れでフィルタリング可能
- [ ] 各タスクに現在の先延ばしスコアが表示される
- [ ] 締切までの残り時間が表示される
- [ ] 新しいタスクが上に表示される（ソート切替可能）

---

## US-008: タスク編集
**As a** タクヤ, **I want to** 登録済みタスクのタイトルや締切を編集したい, **so that** 間違いを修正できる

**Acceptance Criteria:**
- [ ] タスクをタップして編集モードに入れる
- [ ] タイトル、締切日時を変更可能
- [ ] 先延ばし中のタスクは締切変更時に確認ダイアログ表示
- [ ] 変更履歴は保持される（スコア計算の公正性のため）

---

## US-009: タスク削除
**As a** タクヤ, **I want to** 不要になったタスクを削除したい, **so that** 一覧がスッキリする

**Acceptance Criteria:**
- [ ] スワイプまたは長押しで削除オプション表示
- [ ] 確認ダイアログ表示
- [ ] 削除後は一覧から消える
- [ ] スコア確定済みタスクの削除はランキングに影響しない

---

## US-010: タスクの完了報告
**As a** タクヤ, **I want to** タスクを「完了」にしたい, **so that** 先延ばしスコアが確定する

**Acceptance Criteria:**
- [ ] 「完了」ボタンをタップするとスコア確定
- [ ] 確定時のスコアとランキングへのインパクトが表示される
- [ ] 完了アニメーション/エフェクト表示
- [ ] 完了後は先延ばし中一覧から完了一覧に移動

---

# Epic 3: AIタスク定量化

## US-011: AIによるタスク重さ質問
**As a** タクヤ, **I want to** タスク登録後にAIから「何で重いと感じる？」と聞かれたい, **so that** タスクの重さを多次元で認識できる

**Acceptance Criteria:**
- [ ] タスク登録後にAIがチャット形式で質問開始
- [ ] 利害関係者数、金額、期日、難度、未知度、評価への影響の6次元で質問
- [ ] 各次元を1〜5スケールで回答可能
- [ ] スキップも可能（AIがデフォルトスコアを推定）

---

## US-012: AIによる自動タスクスコア算出
**As a** タクヤ, **I want to** AIがタスクの重さスコアを自動算出してほしい, **so that** 先延ばしスコアの基準値がわかる

**Acceptance Criteria:**
- [ ] ユーザー回答 + AIの推定でタスクスコア（1〜100）を算出
- [ ] スコアの内訳が確認可能
- [ ] ユーザーが「これは違う」と思ったら手動調整可能
- [ ] スコアが高いほど先延ばしした時の得点が高い旨を説明

---

## US-013: タスクスコアの表示
**As a** タクヤ, **I want to** タスクのスコアがビジュアルで見えるようにしてほしい, **so that** 直感的に重さが分かる

**Acceptance Criteria:**
- [ ] タスク一覧にスコアバッジ表示
- [ ] 色分け（低:緑、中:黄、高:赤）
- [ ] タスク詳細画面で6次元のレーダーチャート表示

---

# Epic 4: 先延ばしスコアリング

## US-014: リアルタイム先延ばし時間カウント
**As a** タクヤ, **I want to** 先延ばし中のタスクの経過時間がリアルタイムで見えてほしい, **so that** 「今どれだけ先延ばしているか」が分かる

**Acceptance Criteria:**
- [ ] タスク登録時刻からの経過時間をリアルタイム表示
- [ ] 締切までの%消費率も表示
- [ ] バックグラウンドでもカウント継続
- [ ] ダッシュボードに最も先延ばしているタスクを表示

---

## US-015: 先延ばしスコア算出
**As a** タクヤ, **I want to** 「タスクスコア × 先延ばし%消費率」で得点が計算されてほしい, **so that** ギリギリまで伸ばすほど高得点になる

**Acceptance Criteria:**
- [ ] スコア = タスクスコア × (経過時間 / 締切までの総時間) × 100
- [ ] 完了ボタン押下時にスコア確定
- [ ] 期限超過した場合は別カテゴリ（ペナルティなし、ただしボーナスもなし）
- [ ] スコア確定時に演出表示

---

## US-016: 先延ばしスコアの履歴表示
**As a** タクヤ, **I want to** 過去のスコア履歴を確認したい, **so that** 自分の先延ばしパターンが見える

**Acceptance Criteria:**
- [ ] 日別/週別/月別でスコア推移グラフ表示
- [ ] 各タスクのスコア詳細が確認可能
- [ ] 最高スコアのタスクがハイライト表示

---

## US-017: 週次スコア集計
**As a** タクヤ, **I want to** 週次で全タスクの合計スコアが集計されてほしい, **so that** ランキングに反映される

**Acceptance Criteria:**
- [ ] 月曜0:00〜日曜23:59で週次集計
- [ ] 週次合計スコアがランキングに使用される
- [ ] 前週比較が可能
- [ ] 週の途中でも暫定順位が確認可能

---
