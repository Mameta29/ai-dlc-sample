# SABOROU サービス層定義

## サービスアーキテクチャ概要

サービス層はドメインコンポーネント間のオーケストレーションを担当します。
各サービスは複数のコンポーネントを協調させ、ビジネスフローを実現します。

---

## Service 1: TaskLifecycleService

| 項目 | 内容 |
|---|---|
| **目的** | タスクの登録から完了までのライフサイクル管理 |
| **オーケストレーション対象** | TaskComponent, AIComponent, ScoringComponent, AnalyticsComponent |
| **責任** | タスク登録フロー、AI定量化フロー、完了・スコア確定フロー |

### フロー
1. **タスク登録フロー**: Task作成 → AI定量化質問 → スコア算出 → タスクスコア保存
2. **完了フロー**: 完了報告 → スコア確定 → ランキング更新通知 → アナリティクス記録

---

## Service 2: ConversationService

| 項目 | 内容 |
|---|---|
| **目的** | AI対話セッションの管理・データ収集の協調 |
| **オーケストレーション対象** | AIComponent, AnalyticsComponent, NotificationComponent |
| **責任** | 対話開始、メッセージルーティング、データ収集トリガー、定期通知スケジュール |

### フロー
1. **対話開始フロー**: 先延ばしタスク検出 → オープン質問生成 → ユーザー応答待ち
2. **言い訳提示フロー**: パターン生成 → 提示 → 反応記録 → 分析トリガー
3. **煽り対話フロー**: 煽りメッセージ生成 → 反論記録 → 自己像分析トリガー

---

## Service 3: RankingService

| 項目 | 内容 |
|---|---|
| **目的** | ランキングの生成・更新・配信 |
| **オーケストレーション対象** | RankingComponent, ScoringComponent, SocialComponent, NotificationComponent |
| **責任** | スコア確定時のランキング再計算、Realtime配信、週次リセット |

### フロー
1. **スコア確定時**: スコア受信 → ランキング再計算 → Realtime配信 → 順位変動通知
2. **週次リセット**: 全ランキングアーカイブ → リセット → 前週結果通知

---

## Service 4: AnalysisPipelineService

| 項目 | 内容 |
|---|---|
| **目的** | データ分析パイプラインの管理（リアルタイム + バッチ） |
| **オーケストレーション対象** | AnalyticsComponent, AIComponent (分析用) |
| **責任** | リアルタイム簡易分析のトリガー、バッチ分析のスケジュール、自己取扱説明書の更新判定 |

### フロー
1. **リアルタイム分析**: イベント受信 → 簡易集計更新 → 閾値チェック → プロファイル微更新
2. **バッチ分析（日次）**: 日次データ集約 → 8カテゴリ分析実行 → プロファイル更新
3. **取扱説明書更新（週次）**: 全カテゴリ統合 → AI要約生成 → 取扱説明書更新

---

## Service 5: SocialFeedService

| 項目 | 内容 |
|---|---|
| **目的** | フィード生成とソーシャルインタラクション管理 |
| **オーケストレーション対象** | SocialComponent, RankingComponent, NotificationComponent |
| **責任** | フィードアイテム生成、リアクション処理、フォロー時のフィード更新 |

### フロー
1. **フィード生成**: フォロー中ユーザーのアクティビティ集約 → 時系列ソート → ページネーション
2. **アクティビティ発生時**: スコア確定/ランキング変動 → フィードアイテム生成 → フォロワーへ通知

---

## Service 6: OnboardingService

| 項目 | 内容 |
|---|---|
| **目的** | 新規ユーザーのオンボーディングフロー管理 |
| **オーケストレーション対象** | AuthComponent, ProfileComponent, AIComponent, NotificationComponent |
| **責任** | 初回ログイン検出、ニックネーム設定、キャラクター選択、チュートリアル、通知許可 |

### フロー
1. **初回ログイン**: Google認証完了 → ユーザーレコード作成 → ニックネーム入力画面 → キャラクター選択 → チュートリアル → ダッシュボード

---

## Service 7: DataExportService

| 項目 | 内容 |
|---|---|
| **目的** | データエクスポート・自己取扱説明書出力の管理 |
| **オーケストレーション対象** | AnalyticsComponent, ProfileComponent, MCPServerComponent |
| **責任** | JSON全データエクスポート、PDFレポート生成、AI向けプロンプト生成、MCP API Key管理 |

### フロー
1. **全データエクスポート**: データ収集 → JSON整形 → ダウンロードURL生成
2. **PDFレポート**: 分析結果取得 → テンプレート適用 → PDF生成 → ダウンロード
3. **AIプロンプト生成**: プロファイル取得 → マークダウン整形 → クリップボードコピー
