# SABOROU 要件ドキュメント

## インテント分析サマリー

| 項目 | 内容 |
|---|---|
| **ユーザーリクエスト** | 先延ばし行動を肯定・ゲーミフィケーション化し、"自己取扱説明書データ"を蓄積するサービス「SABOROU」の構築 |
| **リクエストタイプ** | 新規プロジェクト（Greenfield） |
| **スコープ見積** | 複数コンポーネント（フロントエンド、バックエンドAPI、AI対話、データ分析、ランキング） |
| **複雑さ見積** | 複雑（AI対話、ゲーミフィケーション、行動データ収集・分析、MCP連携） |
| **品質レベル** | 本番運用前提（品質・スケーラビリティ重視） |

## 機能要件

### FR-01: タスク管理

| ID | 要件 |
|---|---|
| FR-01-01 | ユーザーがタスクを入力できる（タイトル、締切日時を最低限含む） |
| FR-01-02 | 将来的に音声入力対応を想定した入力インターフェース設計 |
| FR-01-03 | タスクの一覧表示・編集・削除が可能 |
| FR-01-04 | タスクの状態管理（先延ばし中 / 完了 / 期限切れ） |

### FR-02: AIタスク定量化

| ID | 要件 |
|---|---|
| FR-02-01 | AIがタスクを多次元的に定量化（利害関係者数、金額、期日、難度、未知度、評価への影響） |
| FR-02-02 | ユーザーに「これって何で重いと感じる？」を多次元で問い、主観的重さスコアを取得 |
| FR-02-03 | タスクスコアの算出と表示 |

### FR-03: 先延ばしスコアリング

| ID | 要件 |
|---|---|
| FR-03-01 | タスクスコア × 先延ばし時間（締切までの%消費率）で得点算出 |
| FR-03-02 | ユーザーが「完了」を押した時点でスコア確定 |
| FR-03-03 | 先延ばし時間のリアルタイム計測・表示 |

### FR-04: ランキング機能

| ID | 要件 |
|---|---|
| FR-04-01 | 週次全タスク総合得点によるグローバルランキング |
| FR-04-02 | フレンド間/グループ内ランキング |
| FR-04-03 | ランキング公開表示（上位者の表彰） |

### FR-05: AI対話（言い訳生成）

| ID | 要件 |
|---|---|
| FR-05-01 | AIがユーザーに「今日やらなくていい理由ある？」とオープンに問いかけ |
| FR-05-02 | AIが3〜4種類の言い訳パターンをランダム提示（言い訳A/B/C/Dテスト） |
| FR-05-03 | ユーザーの反応（同意/反論）をログ記録 |
| FR-05-04 | 言い回しのバリエーション生成（同じ意味で異なる語彙） |
| FR-05-05 | 軽い煽り対話パターン（自己像抽出用） |

### FR-06: データ収集・蓄積

| ID | 要件 |
|---|---|
| FR-06-01 | 受容パターンデータの収集（どの言い訳で安心するか） |
| FR-06-02 | 重さの主観次元データ収集 |
| FR-06-03 | ステークホルダー階層データ収集 |
| FR-06-04 | 自己同一性データ収集（壊したくない自己像） |
| FR-06-05 | 着火閾値データ収集（いつなら動くか） |
| FR-06-06 | 言語的トリガーデータ収集（刺さる単語/効かない単語） |
| FR-06-07 | 回避行動の生体リズムデータ収集（アプリ起動・操作タイミング） |
| FR-06-08 | 自己生成型言い訳データ収集（ユーザー自由記述） |

### FR-07: 自己取扱説明書データ出力

| ID | 要件 |
|---|---|
| FR-07-01 | 蓄積データから「自己取扱説明書」を生成・表示 |
| FR-07-02 | MCP連携で他AIエージェントへ装着可能なデータフォーマットで出力 |
| FR-07-03 | ユーザーが自己取扱説明書を閲覧・ダウンロード可能 |

### FR-08: 認証・ユーザー管理

| ID | 要件 |
|---|---|
| FR-08-01 | Googleソーシャルログインによる認証 |
| FR-08-02 | ユーザープロフィール管理（ニックネーム、アバター） |
| FR-08-03 | フレンド/グループ機能 |

### FR-09: 定期通知・リマインド

| ID | 要件 |
|---|---|
| FR-09-01 | AIからの定期的な「今動く？もうちょっと寝かせる？」プッシュ通知 |
| FR-09-02 | 先延ばしスコアの変動通知 |

## 技術スタック（提案）

| レイヤー | 技術 | 理由 |
|---|---|---|
| フロントエンド | Next.js 16.2 (App Router) + TypeScript | 最新版、~400%高速dev起動、Turbopack、AI対応 |
| UI | Tailwind CSS v4 + shadcn/ui (CLI v4) | AI Agent Skills対応、Design System Presets |
| バックエンドAPI | Next.js Route Handlers + tRPC | 型安全、フロントとの統合容易 |
| データベース | Supabase (PostgreSQL, v1.26.04) | リアルタイム機能、認証統合、RLS、Stripe連携 |
| 認証 | Supabase Auth (Google OAuth) | Googleログイン標準サポート |
| AI | OpenAI API (GPT-4o) | 対話生成、タスク定量化 |
| PBTフレームワーク | fast-check | TypeScript対応、Vitest統合 |
| デプロイ | Vercel (フロント) + Supabase (バックエンド) | Next.js最適化、グローバルCDN |
| プッシュ通知 | Web Push API (Service Worker) | PWA標準機能 |

## データモデル概要

### 主要エンティティ
- **User**: ユーザー情報、認証データ
- **Task**: タスク情報（タイトル、締切、状態、スコア）
- **ProcrastinationSession**: 先延ばしセッション（開始時刻、終了時刻、スコア）
- **AIConversation**: AI対話ログ（メッセージ、パターンタイプ、ユーザー反応）
- **ExcusePattern**: 言い訳パターンデータ（カテゴリ、受容率）
- **UserProfile**: 自己取扱説明書データ（蓄積分析結果）
- **Group**: グループ/フレンド管理
- **Ranking**: ランキングデータ（週次集計）
